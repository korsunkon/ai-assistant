# Устранение проблем

## Проблема: Статус обработки звонка - "error"

### Причина
Whisper требует `ffmpeg` для загрузки аудиофайлов, но на Windows он часто не установлен или не в PATH.

### Решение
Используется библиотека `librosa` для загрузки аудио, которая не требует `ffmpeg`. Это уже исправлено в коде.

### Проверка работы
1. Убедитесь, что `librosa` установлен:
   ```bash
   pip install librosa soundfile
   ```

2. Проверьте логи в `logs/app.log` - там будут детальные сообщения об ошибках

3. Если ошибка повторяется, проверьте:
   - Существует ли файл по пути из БД
   - Достаточно ли места на диске
   - Достаточно ли RAM/VRAM для обработки

## Другие возможные проблемы

### Ollama не отвечает
- Убедитесь, что Ollama запущен: `ollama serve`
- Проверьте модель: `ollama list` (должна быть qwen3:8b)

### Whisper модель не загружается
- Первая загрузка модели может занять время (2-3 GB для large-v3)
- Проверьте интернет-соединение
- Убедитесь, что достаточно места на диске

### Ошибки кодировки файлов
- Убедитесь, что имена файлов не содержат специальных символов
- Используйте латиницу в именах файлов для избежания проблем
